Pull Request
	:특정 브랜치에서 작업한 변경 사항을 다른 브랜치(주로 main 또는 develop)에 병합하겠다는 요청

Collaboration 
	: 프로젝트에서 여러 개발자가 함께 작업하는 과정이나 구조

branches rule
	> Branch name pattern	: 해당 브랜치에서 rule을 적용
	> Require a pull request before merging : 조건(필요 approvals 갯수 지정 등등)
	> Lock branch : 브랜치 추가 생성을 막음


Git에서 origin은 리모트 저장소의 기본 이름
	리포트 저장소 추가 : git remote add origin url

git fetch
	: 리모트 브랜치에 새로운 커밋, 태그, 브랜치 등이 추가된 경우 이를 로컬 저장소로 가져옴
