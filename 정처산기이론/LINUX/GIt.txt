Git
	분산 버전 관리 시스템(DVCS, Distributed Version Control System)
	소스 코드의 변경 사항을 추적하고 여러 명의 개발자가 협업할 수 있도록 돕는 도구
	주로 소프트웨어 개발에서 사용되지만, 버전 관리를 필요로 하는 모든 종류의 프로젝트에 사용 가능

주요 기능
1. 버전 관리:
    소스 코드의 변경 이력을 관리
    이전 버전으로 돌아가거나 특정 버전을 확인

2. 분산형 시스템:
    모든 개발자가 전체 프로젝트의 히스토리를 복사해 로컬 저장소에 저장
    중앙 서버가 없어도 작업가능, 인터넷 연결 없이도 로컬에서 커밋하고 브랜치를 생성

3. 브랜치와 병합:
    독립된 작업을 위해 브랜치를 생성
    브랜치를 이용해 새로운 기능을 개발, 버그를 수정, 완료된 작업을 메인 브랜치에 병합

4. 다양한 협업 워크플로우:
    여러 명의 개발자가 동시에 작업가능
    풀 리퀘스트(Pull Request)를 통해 코드 리뷰와 협업이 가능
 		

		** 제어판-> 사용자 -> 자격증명관리 -> git으로 연결된 데이터 삭제 체크


작동 구조
git add : 변경 사항을 스테이징 영역에 추가
git commit : 스테이징 영역의 변경 사항을 저장소에 저장

설치파일 설정 따로 x
	

.git 파일 내용
HEAD 파일: 현재 작업 중인 브랜치를 가리킴
config 파일: 저장소 설정 정보를 담음
description 파일: 저장소의 설명을 담음
hooks 폴더: Git에서 발생하는 이벤트에 대해 자동으로 실행할 스크립트 파일을 담음.
info 폴더: 저장소 정보를 담고 있는 파일들을 담음
objects 폴더: 모든 파일 버전의 내용을 압축하여 저장
refs 폴더: 브랜치, 태그 등의 참조 정보를 담는 폴더
