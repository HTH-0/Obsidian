OSI 7Layer, TCP/IP 
계층 
1 : 물리 계층 - 케이블, 단자 등등 
2 : 데이터링크 계층 -링크를 기준으로 데이터를 전달할 때, 데이터를 잘 전달하기 위함(본격적인 계층 시작)
3 : 네트워크 - 경로를 결정 (네비게이션 같은 역할)
4 : 전송 - 송, 수신지 간의 데이터 결함여부 체크



1. Network 는 무엇이며 무엇을 목적으로 하는가?
net 와 work를 합친 단어로 데이터 교환을 목적으로 여러 통신장치들을 연결한 통신망이다.


2. Network 의 종류는 크게 ooo/ooo 로 나눌 수 있다
LAN(근거리 통신망)/ WAN(광역 통신망)


3. LAN 의 특징에 대해서 서술하세요
2KM 이내의 근거리 네트워크, 거리가 짧아 비교적 속도가 높다(100Mbps, 1Gbps, 10Gbps).
설치/운영 비용도 상대적으로 저렴하고 설치/유지 보수도 간단하다.
보안이 더 용이하다.


4. WAN의 특징에 대해서 서술하세요
여러 LAN과 다른 유형의 네트워크를 연결.
지리적으로 넓은 지역을 커버(지역, 국가, 대륙)
다양한 전송 기술 사용.
비용이 높고 관리하기 위해 높은 전문 지식, 복잡한 네트워크 관리가 요구된다.


5. InterNet 이란
Inter + Net
전세계적으로 연결된 네트워크의 집합.


6. 프로토콜은 무엇인가?
네트워크 환경에서 데이터 통신을 위한 규칙/규약
구문, 의미, 타이밍 의 요소로 나뉜다.


7. 네트워킹 모델의 목적은 무엇인가?
통신망 구현을 위한 지침서
효율적인 네트워크 설계를 위한 모델.
장애가 발생 시, 문제 원인 탐색이 용이하다.


8. TCP/IP 모델과 OSI 7 모델의 계층을 적고 각 계층별 역할을 적으세요
OSI 7 모델 / [TCP/IP모델]
1- 물리 계층 : 장치 연결, 신호세기 유지 , 전기적 신호를 데이터로 주고받는 역할(허브, 리피터, 통신 케이블)
2- 데이터링크 계층 : 장치 간 데이터 전송 관련(브릿지, 스위치)
3- 네트워크 계층 : 경로 탐색 관련(라우팅이라고 함)(라우터, 공유기, L3스위치)
4- 전송 계층 : 호스트 간의 데이터 전송 관련(게이트웨이, 로드밸런서), 송/수신 간의 데이터 신뢰성을 확보.

5- 세션 계층 : 통신 연결을 관리, 손상된 경우 연결 복구를 시도.
6- 프레젠테이션 계층 : 서로 다른 사용자 시스템 간에 번역을 담당.
7- 애플리케이션 계층 : 직접적인 서비스(웹 브라우저, 메일)
: APP에서의 작업 관련 [애플리케이션 계층]

신호 : 부호(code)-의미를 가지는 기호

이더넷 세부 x



1 물리계층 하는일/역할/관련 장치는?
실제 장치들을 연결하기 위해 필요한 전기적, 물리적 세부 사항
역할 : 장치의 물리적 연결
약속(프로토콜) : 전기적 신호 전달(PUD : Bit)
장치 : 케이블, RJ45(단자), 허브, 리피터

2 Cable 이란
네트워크 장비나 장치를 연결하는데 사용되는 물리적 매체
주로 구리선, 네트워크에서 데이터 전송을 담당

3 LAN Cable 종류
통신기기 사이에 정보를 주고받기 위한 케이블(STP, UTP-많이 사용됨, FTP)

4 부호(Code) 란?
의미를 가지는 약속된 기호, 디지털 부호 : 컴퓨터가 이해하는 부호(0,1 이산데이터)

5 신호(Signal)이란?
원거리로의 부호 전달 형태

6 신호전달에 필요한 요소?
통신망, 통신기기, 에너지원
부호 전달하기 위한 에너지 : 전기에너지
부호 전달하기 위한 인프라 : 통신망

7 리피터란?
신호 증폭 장치

8 허브란
집선 장치
여러 대의 컴퓨터 간 통신이 가능
리피터처럼 신호를 증폭시키는 기능을 가지고 있음
발송지 데이터가 나머지 모든 컴퓨터에 전달됨
